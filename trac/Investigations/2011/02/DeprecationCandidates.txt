As we work towards a cleaner API (see
`Omero/GatewayApi </ome/wiki/Omero/GatewayApi>`_ ) we will be looking to
reduce the number of services and methods available. This process will
involve consultation with many users of the existing API. This page is
part of that consultation and will be used to list potential services
and methods that can be deprecated. Also see
`Api/DeprecatedServices </ome/wiki/Api/DeprecatedServices>`_ for
functionality that has already been deprecated.

There are several potential strategies for identifying unused
methods/services. To get started, I will grep
`OmeroPy </ome/wiki/OmeroPy>`_ for use of each method... Below are
listed Stateless services only. A similar list for Stateful services can
be found at
`Api/DeprecationCandidatesStateful </ome/wiki/Api/DeprecationCandidatesStateful>`_.

Only used in a couple of places.

::

    sharedResources().repositories()
    sharedResources().getTable(file)

populate\_roi.py and populate\_metadata.py do this:

::

    sr = self.service_factory.sharedResources()
    self.table = sr.newTable(1, '/%s.r5' % name)

processor.py

::

    session.sharedResources().addProcessor(prx)
